# How to Contribute

We'd love to accept your patches and contributions to this project. 

## Contribution Process

1.  **Issue Discussion:** If you plan to implement a new feature or change existing behavior, please open an issue first to discuss your proposal with the maintainers. This ensures your time is well-spent and your contribution aligns with the project's direction.
2.  **Pull Requests:**
    *   Fork the repository and create your branch from `main`.
    *   If you've added code that should be tested, add tests.
    *   If you've changed APIs, update the documentation.
    *   Ensure the test suite passes.
    *   Make sure your code lints.
3.  **Review:** Once your PR is submitted, it will undergo a review by the maintainers. They may provide feedback or request changes before merging.

## Code Style

Please follow the existing code conventions in the repository:
*   **Python:** formatted with `black`, linted with `ruff`.
*   **Javascript/CSS:** Please adhere to general clean code practices and the existing formatting.

## Community Guidelines

This project follows Google's Open Source Community Guidelines. Please be respectful and considerate of others when interacting in issues and pull requests.
